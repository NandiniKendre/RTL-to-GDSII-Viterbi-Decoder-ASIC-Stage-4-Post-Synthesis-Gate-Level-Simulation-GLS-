# RTL-to-GDSII-Viterbi-Decoder-ASIC
# Stage 4.1: Zero Delay Gate-Level Simulation (GLS)

---

## Overview

This repository documents **Stage 4.1: Zero Delay Gate-Level Simulation (GLS)** of the RTL-to-GDSII ASIC implementation flow for a **K=7 Convolutional Encoder and Viterbi Decoder**.

Zero Delay Gate-Level Simulation is the first level of post-synthesis verification performed after Logic Synthesis. In this stage, the synthesized gate-level netlist is simulated using standard cell library models while ignoring propagation delays.

The primary objective is to verify that the synthesized gate-level implementation preserves the functional behavior of the original RTL design and produces identical outputs for all verification test cases.

This stage serves as an important validation checkpoint before introducing timing effects through Unit Delay and SDF-Annotated simulations.

---

## Objectives

- Import synthesized gate-level netlist.
- Load standard cell simulation models.
- Perform gate-level functional verification.
- Verify functional equivalence between RTL and synthesized netlist.
- Validate Viterbi Decoder functionality.
- Compare gate-level outputs against RTL reference outputs.
- Generate simulation waveforms.
- Confirm design readiness for advanced timing-aware verification.

## Zero Delay GLS Flow

### 1. Netlist Import

Imported the synthesized gate-level netlist generated during Logic Synthesis.

### 2. Standard Cell Library Loading

Loaded standard cell simulation models required for gate-level verification.

### 3. Testbench Execution

Applied the RTL verification environment to the synthesized netlist.

Verification included:

- Decoder output validation
- State transition verification
- Survivor path verification
- Path metric verification
- Functional correctness checks

### 4. Zero Delay Simulation

Performed gate-level simulation without propagation delays.

Characteristics:

- No cell delays
- No interconnect delays
- Pure functional verification
- RTL-equivalent timing behavior

### 5. Output Comparison

Compared synthesized netlist outputs with RTL reference outputs.

### 6. Verification Completion

Confirmed functional equivalence and successful simulation execution.

---

## Verification Methodology

The synthesized gate-level netlist was simulated using the same testbench environment used during RTL verification.

The verification process focused on:

- Functional correctness
- Decoder output accuracy
- State machine integrity
- Survivor path selection
- Branch metric calculations
- Path metric calculations

All outputs generated by the synthesized netlist were compared against expected RTL results.

## Advantages of Zero Delay GLS

- Fast simulation runtime
- Early netlist verification
- Functional equivalence checking
- Identification of synthesis-induced issues
- Low simulation complexity
- Preparation for timing-aware simulations

---

## Limitations

Zero Delay GLS does not model:

- Gate propagation delays
- Interconnect delays
- Setup violations
- Hold violations
- Timing-related glitches

These effects will be analyzed in:

- Stage 4.2: Unit Delay GLS
- Stage 4.3: SDF Annotated GLS
